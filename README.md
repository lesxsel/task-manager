# 📉 Комната №213: Кривенькая Канбан-доска на Django + React

Учебный, откровенно простенький и пока не до конца допиленный пет-проект. Создавался с нуля ради того, чтобы пощупать связку бэкенда и фронтенда, пережить капризы Windows и понять, как работает роутинг. 

Никакого продакшн-реди архитектуры тут нет. Это просто легаси-код в процессе сборки.

---

## 🛠️ Что тут есть (и как оно еле-еле работает)

*   **Бэкенд (Django + DRF):** Самая базовая админка и простейшие эндпоинты, которые отдают JSON. Код сырой, авторизации пока нет (дверь открыта для всех), а БД крутится на дефолтном SQLite.
*   **Фронтенд (React + TypeScript):** Самая простая верстка на Tailwind 4. Подключена библиотека `@hello-pangea/dnd` для перетаскивания.
*   **Главный косяк на сегодня:** Карточки на экране двигаются, но из-за кривизны запросов PATCH статус в базе Django после перезагрузки страницы слетает обратно. Это не баг, это фича в процессе отладки.

---

## 📊 Модель данных

Всего три базовые таблицы в стиле «я у мамы архитектор»:
1.  **Workspaces** — рабочая папка (в нашем случае «Комната №213»).
2.  **Projects** — проект внутри папки.
3.  **Tasks** — сами карточки задач с тремя статусами и тремя уровнями паники (приоритета).

---

## 🔧 Как запустить это чудо (если очень хочется)

### 1. Сервер (Django)
```bash
cd backend
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install django djangorestframework django-cors-headers
python manage.py migrate
python manage.py runserver
```

### 2. Интерфейс (React)
```bash
cd frontend
npm install
npm run dev
```
